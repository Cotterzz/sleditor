Distributing Your Library via CDN
There are several approaches depending on your needs:

Option 1: Use jsDelivr (Easiest)
If your library is on npm or GitHub, jsDelivr serves it automatically:

html
<!-- From npm -->
<script src="https://cdn.jsdelivr.net/npm/your-package@version/dist/file.js"></script>

<!-- From GitHub -->
<script src="https://cdn.jsdelivr.net/gh/user/repo@version/file.js"></script>
Steps:

Publish to npm: npm publish
Use the jsDelivr URL immediately (no signup needed)
Option 2: unpkg (npm packages)
html
<script src="https://unpkg.com/your-package@version/dist/file.js"></script>
Just publish to npm and it's available instantly.

Yes, that's correct!
GitHub Pages (Direct)
Soft limit: 100GB bandwidth/month
Build limit: 10 builds/hour
GitHub may throttle or disable your site if exceeded
jsDelivr (using GitHub as source)
jsDelivr caches your files on their own CDN infrastructure
After the first request, traffic is served from jsDelivr's edge nodes, not GitHub
No practical rate limits for public repos
Direct GitHub Pages:
User â”€â”€â–º GitHub (limited bandwidth)

Via jsDelivr:
User â”€â”€â–º jsDelivr CDN (cached) â”€â”€â–º GitHub (only on cache miss)
Summary
GitHub Pages	jsDelivr (GitHub source)
Bandwidth limit	~100GB/month	Effectively none
Global edge nodes	Limited	750+ PoPs
Caching	Basic	Aggressive CDN caching
Cost	Free	Free
For a library you're distributing: jsDelivr is the better choice even though both use GitHub as the origin. You get proper CDN benefits without worrying about GitHub's limits.
